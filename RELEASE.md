# Release Process

This project uses semantic versioning.

## Versioning

- `v0.1.0` is the initial release baseline.
- Increment patch versions for fixes.
- Increment minor versions for new features.
- Increment major versions for breaking changes.

## Suggested release flow

1. Update `CHANGELOG.md`.
2. Bump `VERSION`.
3. Commit the release prep.
4. Tag the release as `vX.Y.Z`.
5. Push the tag and publish a GitHub Release.
