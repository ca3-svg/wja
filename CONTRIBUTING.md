# Contributing

Thanks for helping improve `wja`.

## Workflow

1. Open an issue or describe the change you want.
2. Keep changes focused and easy to review.
3. Update docs when behavior changes.
4. Use clear commit messages.

## Suggested conventions

- Branch names: `feature/<topic>`, `fix/<topic>`, `docs/<topic>`
- Commits: short, imperative, and specific
- Releases: semantic versioning with `vMAJOR.MINOR.PATCH`
